# **Payment Simulator**

Go, PostgreSQL, Apache Kafka

Финтех-проект на Go для асинхронной обработки платежей с антифрод-проверками.  
Симулятор переводов между кошельками с использованием событийно-ориентированной архитектуры.

---

## **О проекте**

Это pet-проект, созданный для демонстрации навыков разработки на Go в контексте финтех-задач.  
Основные возможности:

- ✅ Создание и управление кошельками
- ✅ Асинхронные переводы через Kafka
- ✅ Антифрод-проверки (лимиты, частотность, переводы себе)
- ✅ Атомарные транзакции в PostgreSQL (`FOR UPDATE`)
- ✅ Graceful shutdown
- ✅ Полностью докеризирован

---

## **Архитектура**

```
[Клиент] → [API] → [Kafka] → [Worker] → [PostgreSQL]
              ↑                    ↓
              └────────────────────┘
         (асинхронная обработка)
```

- **API** — принимает запросы, сохраняет транзакцию, отправляет событие в Kafka
- **Worker** — читает события, проверяет фрод, выполняет перевод
- **PostgreSQL** — гарантирует целостность данных через транзакции

## **Быстрый старт**

### Предварительные требования
- Docker и Docker Compose
- Go 1.21 (опционально, для локального запуска)
- Make (для удобства)

### Запуск одной командой
```bash
# Клонировать репозиторий
git clone https://github.com/yourusername/payment-sim.git
cd payment-sim

# Запустить всё (БД, Kafka, API, worker)
make run-dev
```

### Или пошагово
```bash
# 1. Поднять инфраструктуру
make docker-up

# 2. Накатить миграции
make migrate-up

# 3. Запустить API (в отдельном терминале)
make run-api

# 4. Запустить worker (в отдельном терминале)
make run-worker
```

Сервер будет доступен на `http://localhost:8080`

## **API Endpoints**

### Кошельки

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `POST` | `/api/wallets` | Создать новый кошелёк |
| `GET` | `/api/wallets/{id}/balance` | Получить баланс |
| `GET` | `/api/wallets/{id}/transactions?limit=10` | История операций |

| Метод | Endpoint | Описание |
|-------|----------|----------|

## **Примеры запросов**

### Создать кошелёк
```bash
curl -X POST http://localhost:8080/api/wallets \
  -H "Content-Type: application/json" \
  -d '{"balance": 1000000}'
```
**Ответ:**
```json
{
  "id": "123e4567-e89b-12d3-a456-426614174000",
  "balance": 1000000,
  "created_at": "2026-03-17T12:00:00Z"
}
```

### Перевести деньги
```bash
curl -X POST http://localhost:8080/api/transactions \
  -H "Content-Type: application/json" \
  -d '{
    "from_wal_id": "123e4567-e89b-12d3-a456-426614174000",
    "to_wal_id": "987fcdeb-51a2-43f1-bc8d-123456789012",
    "amount": 50000,
    "description": "За услуги"
  }'
```
**Ответ:**
```json
{
  "id": "tx-uuid",
  "status": "pending",
  "amount": 50000,
  "created_at": "2026-03-17T12:00:00Z"
}
```

### Проверить статус транзакции
```bash
curl http://localhost:8080/api/transactions/tx-uuid
```
**Ответ:**
```json
{
  "id": "tx-uuid",
  "status": "approved",
  "amount": 50000,
  "created_at": "2026-03-17T12:00:00Z"
}
```

### История кошелька
```bash
curl "http://localhost:8080/api/wallets/123e4567-e89b-12d3-a456-426614174000/transactions?limit=5"
```
**Ответ:**
```json
{
  "id": "wallet-uuid",
  "transactions": [
    {
      "id": "tx-uuid",
      "type": "outgoing",
      "counterparty": "987fcdeb-51a2-43f1-bc8d-123456789012",
      "amount": 50000,
      "status": "approved",
      "description": "За услуги",
      "created_at": "2026-03-17T12:00:00Z"
    }
  ]
}
```

## **Антифрод правила**

Перед выполнением перевода worker запускает проверки:

| Правило | Описание | Действие |
|---------|----------|----------|
| **AmountRule** | Сумма > 1_000_000 | `fraud` |
| **FrequencyRule** | > 10 транзакций за 5 минут | `fraud` |
| **SelfTransferRule** | Перевод самому себе | `fraud` |

При обнаружении подозрительной операции статус меняется на `fraud`, деньги не переводятся, в `error_msg` записывается причина.

## **Интеграционные тесты**

Проект содержит интеграционные тесты с использованием [Testcontainers](https://golang.testcontainers.org/).

```bash
# Запустить интеграционные тесты
go test ./test/integration/... -v -timeout 60s
```

Что проверяется:

✅ Успешный перевод между кошельками

✅ Обработка недостатка средств

✅ Обработка несуществующих кошельков

✅ Атомарность транзакций

Тесты поднимают реальный PostgreSQL в Docker-контейнере, выполняют операции и проверяют результаты.
